# About Amplipost

## 一句话介绍

**Amplipost** = **Amplify** + **Post**。一句话输入，四个平台输出，全程无需人工干预的智能内容营销中台。

---

## 起源

做营销内容最费时的不是写本身，而是**多平台适配 + 反复发布**。同一个内容要改标题、调格式、避违禁词、配图、处理平台规则——发一次要折腾半小时起步。

Amplipost 最初是想解决一个具体问题：**能不能让 AI 帮我全自动发完小红书再发抖音再发闲鱼，我只需要说一句话？**

在这个过程中，逐渐演变成了一套完整的 Multi-Agent 内容生产体系。

---

## 核心理念

### 1. 职责分离

传统自动化脚本让同一个 Agent 既生成内容又做质量判断——这有天然的偏差。Amplipost 的解法是三个 Agent 各司其职：

- **content-coordinator**：生成内容、调度流程
- **content-reviewer**：独立质量评审（用统一标准打分，不对生成方手下留情）
- **publish-guard**：风控评估（频率、间隔、多样性）

三者通过结构化 JSON 通信，互不干涉，形成真正的制衡机制。

### 2. 自主决策，不打断用户

系统只在**两种情况**下询问用户：登录态失效（物理限制）、指令完全歧义（无法推断主题）。其他所有决策——平台选择、内容重写、违禁词替换、风控延迟——全部自主完成。

### 3. 越用越聪明

`memory.md` 记录了历史发布数据、内容方向、用户偏好和风控日志。随着使用次数增加，Agent 会逐渐理解你的风格和平台规律，输出质量持续提升。

## 技术栈

| 组件 | 技术 | 用途 |
|------|------|------|
| 核心架构 | Claude Code Multi-Agent | 内容生成、调度、评审、风控 |
| 浏览器自动化 | Playwright | 各平台网页操作 |
| 反爬增强 | Scrapling | 应对平台反爬机制 |
| Hooks | Python + Claude Code hooks | 发布前合规检查、发布后结果验证 |
| 图片生成 | GPT-4o + PIL | 抖音信息图自动生成 |

---

## 支持的平台

| 平台 | 定位 | 内容类型 |
|------|------|---------|
| 闲鱼 | 商品变现 | 二手商品、数码设备 |
| 小红书 | 种草分享 | 干货、经验、好物推荐 |
| B站专栏 | 深度内容 | 技术分析、教程、评测 |
| 抖音图文 | 流量获取 | 短资讯、热点话题 |

---

## 与其他工具的区别

| 能力 | Amplipost | 普通自动发布脚本 |
|------|-----------|----------------|
| 内容生成 | AI 自主生成，多平台独立适配 | 需手动准备 |
| 质量评审 | 独立 Agent 评审，不通过自动重写 | 无 |
| 风控拦截 | publish-guard 评估频率/多样性 | 无 |
| 违禁词处理 | 生成时自动替换 | 需人工检查 |
| 自主决策 | 全程无需确认 | 需中途人工确认 |
| 长期记忆 | memory.md 持续积累经验 | 无 |
