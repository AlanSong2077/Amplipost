# 小红书内容生成规则（通用框架）

## 一、平台定位与内容策略

### 小红书核心特征
- **字数**：200-300 字（不含标签）
- **风格**：碎片化、种草感、情绪化、有温度
- **用户**：泛用户，年轻消费群体为主
- **目的**：引发兴趣、种草、引流私信
- **结构**：情绪化开场 + 短干货 + 互动引导

### 用户画像（通用）
- 18-35 岁，女性偏多（但泛用户均可）
- 喜欢真实、有温度的内容
- 对「攻略」「教程」「测评」「种草」类内容敏感
- 容易被情绪共鸣打动

---

## 二、信息收集阶段

收到用户请求后，先明确以下信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| **主题** | 帖子核心话题 | 某个产品体验、某个行业分析、某个教程 |
| **目的** | 用户想达到什么效果 | 种草安利、引流咨询、分享干货 |
| **痛点** | 目标读者的困惑/担忧 | 不知道如何选择、担心效果、不知道是否值得 |
| **受众** | 目标读者画像 | 职场人、学生、宝妈等 |

若用户只提供了部分信息，应根据已有信息合理推断，**不要反复追问**。

---

## 三、热点搜索策略

使用 `web_search` 搜索以下内容，选取 1-2 个相关热点融入内容：

```
搜索词模板：
- "小红书 热门话题 2025"
- "小红书 X 领域 爆款笔记"
- "小红书 热门标签 趋势"
```

**热门标签参考：**

| 场景 | 推荐标签 |
|------|---------|
| 职业发展 | #职场 #自我提升 #职业规划 #打工人 #职场干货 |
| 科技数码 | #科技 #AI #数码 #编程 #科技改变生活 |
| 教育学习 | #学习 #自我提升 #干货 #教程 #学习方法 |
| 生活方式 | #生活 #日常 #记录 #分享 #生活小技巧 |
| 理财 | #理财 #投资 #存钱 #财务自由 #省钱 |

## 四、标题生成规则

**字数：≤ 20 字**

**高转化标题公式：**

1. **数字干货型**：`X个/X步 + 核心关键词 + 结果`
   - 例：`3步搞定XXX，小白也能轻松上手`

2. **痛点解答型**：`常见疑问 + 答案揭晓`
   - 例：`XXX真的很值得吗？答案来了`

3. **亲历分享型**：`我/真实/亲测 + 结果`
   - 例：`亲测有效！我是怎么做到XXX的`

4. **对比反差型**：`A vs B 哪个更好`
   - 例：`XXX vs XXX，真实对比告诉你怎么选`

5. **种草安利型**：`真香/强烈推荐 + 产品/方法`
   - 例：`这个XXX真的太香了，后悔没早用`

**避免：**
- 夸大承诺词汇
- 明显广告感
- 全大写、过多感叹号
- 任何 emoji / 表情符号（正文和标题均不得出现）

## 五、正文生成规则

### 字数要求
- **正文：200-300 字**（不含标签）
- 超出 300 字需精简，低于 200 字需补充细节

### 结构模板

```
【开场：痛点共鸣或场景引入】（30-50字）
用第一人称描述目标读者的真实痛点或场景，引发共鸣。
口语化，有情绪，像在跟朋友说话。

【核心干货：有价值的内容】（120-160字）
提供真实有价值的内容：
- 具体步骤/方法/建议
- 真实数据或体验
- 有用的技巧或洞见
包含具体信息，避免广告腔。

【自然收尾】（30-50字）
轻描淡写地提及目的或服务，以帮助者姿态出现，不强推。

【收尾：互动引导】（20-30字）
以问句或邀请结尾，引导评论互动。

【话题标签】（单独一行）
#标签1 #标签2 #标签3
```

### 写作要点

**✅ 要做的：**
- 用第一人称「我」，偶尔用「咱」「宝子」「同学」
- 加入真实情绪词：「真的没想到」「太香了」「绝了」
- 用省略号表示停顿
- 加入具体数字
- 句子长短交替，有节奏感

**❌ 要避免的：**
- 「首先、其次、最后」等列举式结构
- 「综上所述」「总的来说」等 AI 感词汇
- 夸大承诺：「保证」「绝对」「100%」
- 虚假数据
- 连续使用感叹号 `！！！`
- **任何 emoji / 表情符号**

### 示例对比

**❌ 广告感强的版本：**
> 我们机构专业提供XXX服务，成功率高达95%！我们有专业团队，一对一辅导！现在咨询还有优惠！

**✅ 诚恳专业的版本：**
> 上个月有个朋友问我，XXX到底值不值得……
>
> 说实话，这个问题没有标准答案。但如果你也在纠结，我可以分享一些自己的真实感受。
>
> 关键是要看你的需求是什么，不同情况适合不同的方案。
>
> 有类似困惑的朋友可以评论区聊聊，或者来找我聊聊～
>
> #XXX #干货分享 #经验分享

## 六、内容确认流程

生成内容后，按以下格式展示给用户确认：

```
📝 **生成的小红书笔记**

**标题：**
{标题}

**正文：**
{正文}

---
字数：{正文字数} 字
融入热点：{使用的热点标签}

是否直接发布？还是需要修改某些部分？
```

用户确认后，将标题和正文传入发布脚本。

---

## 七、多轮修改处理

| 用户反馈 | 处理方式 |
|---------|---------|
| 「太像广告了」 | 减少推广植入，增加干货比例，改用更自然的提及方式 |
| 「太像 AI 了」 | 增加口语化表达、加入具体细节、打乱句式结构 |
| 「太长了」 | 压缩到 200 字，删除过渡句，保留核心内容 |
| 「换个标题」 | 用不同公式重新生成 3 个标题供选择 |
| 「更专业一点」 | 增加具体数据、步骤说明 |
| 「更接地气」 | 增加口语词、真实场景描述 |

---

## 八、合规要求

### 禁止内容
- 极限词：最好、最强、第一、唯一、100%、绝对、保证
- 直接导流：微信号、QQ号、手机号、二维码
- 站外链接：点击链接、扫码等
- 虚假宣传：编造数据、夸大效果

### 合规替代表达
| 违规写法 | 合规替代 |
|---------|---------|
| 「最好」 | 「很不错」「广受好评」 |
| 「保证效果」 | 「很多人反馈效果不错」 |
| 「100%成功」 | 「成功率较高」「不少人都做到了」 |
| 「加微信xxxx」 | 「评论区留言」「私信我」 |
