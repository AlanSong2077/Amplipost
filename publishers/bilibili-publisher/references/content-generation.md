# B站专栏内容生成规则（通用框架）

## 一、平台定位与内容策略

### B站 vs 小红书的核心差异

| 维度 | 小红书 | B站专栏 |
|------|--------|---------|
| 字数 | 200-300 字 | 800-1500 字 |
| 风格 | 碎片化、种草感 | 深度、干货、有逻辑 |
| 用户 | 泛用户、女性偏多 | 25-35岁理性用户 |
| 目的 | 引发兴趣、引流私信 | 建立专业权威、长尾搜索 |
| 结构 | 情绪化开场 + 短干货 | 完整论述 + 数据支撑 |

### B站用户画像（通用目标受众）
- 25-35 岁，理性消费者 / 职场人士 / 专业人士
- 有学习、提升需求，理性决策，不喜欢被推销
- 喜欢有数据、有逻辑、有真实案例的内容
- 会主动搜索相关关键词

---

## 二、内容生成流程

收到用户请求后：

1. 确定目标人群和核心话题
2. 用 `web_search` 搜索相关热点和数据（可选）
3. 按下方规则生成标题 + 正文
4. 展示给用户确认
5. 调用 `bilibili_publish.py` 发布

---

## 三、标题生成规则

**字数：≤ 40 字（B站专栏标题可以比小红书长）**

**高转化标题公式：**

1. **深度分析型**：`主题 + 核心问题 + 完整指南/深度解析`
   - 例：`XXX 申请全攻略：从评估到拿 offer 的完整流程`

2. **数据驱动型**：`用数据说话 + 核心结论`
   - 例：`花 X 万读 XXX，毕业后薪资涨了多少？真实数据分析`

3. **对比决策型**：`A vs B，帮你做决定`
   - 例：`XXX vs 国内考研：职场人该怎么选？深度对比`

4. **经验分享型**：`亲历 + 完整过程 + 结果`
   - 例：`我是如何在职完成 XXX 并实现职业转型的`

5. **避坑指南型**：`X 个坑 / 误区 + 解决方案`
   - 例：`申请 XXX 前必看：这 5 个误区让大多数人白费力气`

**避免：**
- 标题党、夸大承诺
- 任何 emoji / 表情符号
- 虚假承诺词汇

---

## 四、正文生成规则

### 字数要求
- **正文：800-1500 字**（B站专栏适合深度内容）
- 分段清晰，每段 100-200 字
- 可使用小标题（## 或加粗）分隔章节

### 结构模板

```
【引言：场景引入或痛点共鸣】（100-150字）
用具体场景或数据开场，引发目标读者共鸣。

【第一章：核心问题分析】（200-300字）
深入分析读者最关心的问题，提供有价值的视角。
必须包含：具体数据、官方信息、真实案例（匿名）

【第二章：干货内容】（300-500字）
提供可操作的建议、步骤、清单。
可以用小标题、加粗关键词提升可读性。

【第三章：常见误区或注意事项】（150-200字）
指出读者容易犯的错误，体现专业深度。

【结尾：互动引导】（100-150字）
引导读者评论互动（「你觉得……？欢迎评论区聊聊」）
可以自然提及服务或产品，但不强制推销
```

### 写作要点

**B站专栏特有要求：**
- 逻辑严密，有论点有论据，不能只有情绪
- 数据要有来源感（「根据官方数据」「据我们服务的客户」）
- 可以适当使用 Markdown 格式（加粗、小标题）提升可读性
- 段落之间要有过渡，不能像列表一样堆砌
- 结尾引导读者评论互动

**禁止：**
- 任何 emoji / 表情符号（正文和标题均不得出现）
- 「首先其次最后」等机械列举
- 「综上所述」「总的来说」等 AI 感词汇
- 夸大承诺：「保录」「100%成功」「绝对」
- 虚假数据
- **B站平台特有禁止项**（详见 `bilibili-rules.md`）

## 五、话题选择规则

根据目标人群自行确定话题方向，确保每篇内容聚焦一个核心话题。

### 用户指定话题时
直接使用用户指定的话题方向。

### 用户未指定话题时（自动选择）
按以下优先级依次轮换，避免连续发布同类话题：

| 优先级 | 话题类别 | 原因 |
|--------|---------|------|
| 1 | 行业洞察类 | 受众广，搜索量大 |
| 2 | 性价比对比类 | 理性用户喜欢数据对比 |
| 3 | 职业发展类 | 职场人最关心的问题 |
| 4 | 入门指南类 | 新手刚需，高价值引流 |
| 5 | 真实案例类 | 有说服力，转化率高 |

**轮换规则：** 每次自动选择时，从优先级 1 开始，若上一篇已用该类别则顺延到下一类别，形成循环。

## 六、标签推荐

每篇文章建议添加 5-8 个标签，根据具体内容调整：

| 场景 | 推荐标签 |
|------|---------|
| 职业发展 | 职业规划、职场、技能提升、自我提升 |
| 教育留学 | 留学、在线教育、学历提升、研究生 |
| 金融理财 | 理财、投资、资产配置、财务自由 |
| 科技数码 | 科技、数码、AI、人工智能、编程 |

---

## 七、内容确认流程

生成内容后展示给用户确认：

```
**B站专栏草稿**

**标题：**
{标题}

**正文预览（前200字）：**
{正文前200字}...

---
字数：{正文字数} 字
推荐标签：{标签列表}

是否直接发布？还是需要修改？
```

---

## 八、多轮修改处理

| 用户反馈 | 处理方式 |
|---------|---------|
| 「太短了」 | 扩展到 1200-1500 字，增加案例和数据 |
| 「太像广告」 | 减少服务植入，增加干货比例 |
| 「太学术了」 | 增加口语化表达，加入真实场景 |
| 「换个角度」 | 换一个话题方向重写 |
| 「加更多数据」 | 补充具体数字和数据支撑 |
